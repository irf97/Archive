Argos Automated Research Spec

A local-first workflow for turning chats into durable research memory: source-backed domain knowledge, reviewable claims, executable outcomes, and agent jobs that never promote unsupported text into knowledge.

System: ENS Navigator / Argos
Storage: local JSON via server.mjs
Primary rule: no unsourced claims in memory
Status: implementation spec
Goal Principles Research Flow Objects APIs Agent Contract Gates Roadmap

Goal

Argos should use Codex as an automated research operator that improves memory from chats without confusing raw chat content, model output, and researched knowledge.

Stable memory is not a summary. Stable memory is a set of claims, sources, questions, decisions, and outcomes with explicit provenance and review status.
research backed local first human reviewable no orphan claims

Operating Principles

Separate raw chat from memory

Chats enter as intake records. They can propose hypotheses, constraints, or questions, but they do not become approved knowledge directly.

Research before promotion

Claims need sources. Sources should prefer primary literature, standards, official docs, datasets, or public institutional records.

Outcomes are executable

Domain Knowledge should produce tasks, dependency graphs, decision logs, and validation plans, not just long notes.

Research Flow

1Chat Intake

Capture raw chat, project, timestamp, user intent, and candidate domain labels.

2Extraction

Split chat into hypotheses, claims, questions, constraints, and decisions.

3Research Job

Search sources for each candidate item. Unsupported items stay as hypotheses or needs-source tasks.

4Ingest

Write structured sources, claims, questions, links, and people into Domain Knowledge.

5Review

Move items through Inbox, Needs Source, Approved, Disputed, Rejected.

6Outcomes

Create task graphs with evidence gates and dependencies.

Codex Agent Contract

Extractor

Turns chats into candidate objects. Must label unsupported ideas as hypotheses or questions.

Researcher

Finds primary sources, records URLs, extracts minimal evidence notes, and returns structured packets.

Pressure Tester

Searches for contradictions, weak assumptions, missing controls, and boundary conditions.

Outcome Builder

Creates task graphs from approved claims, open questions, disputed items, and needs-source items.

Memory Promoter

Promotes only reviewed, sourced, stable items into long-term Argos memory.

Auditor

Reports orphan claims, stale sources, broken links, uncited outcome tasks, and inconsistent statuses.

Quality Gates

GatePass conditionFail action
Source gateClaim has at least one source ID and source URL.Move to needs_source.
Primary-source gateSource type is paper, RFC, official doc, dataset, or institutional page.Keep as hypothesis or low-confidence claim.
Contradiction gateNo approved contradictory claim without a dispute link.Create disputed review task.
Outcome gateTask has evidence and completed dependencies.Block movement to Done.
Memory gateApproved status, evidence, provenance, and no unresolved contradiction.Do not promote to stable memory.

Implementation Roadmap

  1. Add chat-intake.json and research-jobs.json stores beside current data files.
  2. Add server endpoints for intake, job creation, job status, and promotion audit.
  3. Add an Argos Memory page or subtab showing raw chat candidates, extracted hypotheses, and promotion status.
  4. Add a deterministic audit runner: orphan claims, missing sources, stale source URLs, uncited outcomes, unresolved disputes.
  5. Wire Codex job execution with strict output schema and provenance labels.
  6. Make Outcomes able to spawn research jobs from blocked tasks and write results back into Domain Knowledge.
