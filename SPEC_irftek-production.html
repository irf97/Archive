IrfTek Production Specification

IrfTek is a local-first semantic runtime substrate for humans, devices, places, personas, tools, AI agents, and local domains. It is an authority-resolved object runtime: semantic objects hold state, local authorities own truth, peers propagate state, projection surfaces render it, AI reads it, memory compacts it, and only necessary facts anchor permanently.

local-firstcloud-optional proximity-governedAI-readable peer-distributedoffline-capable anti-bloatsemantic runtime
Absolute reduction: local truth + peer spread + semantic memory + AI-readable reality.

2. Core Philosophical Principles

Local truth over global truth

Authority belongs near the object, place, person, device, or domain being coordinated.

Semantic state over raw telemetry

Raw streams are temporary. Meaning, state change, summaries, lineage, and proofs are retained.

Exploration over feeds

Objects appear through local discovery, domain interest, permission, and intentional traversal.

Personas over universal accounts

Humans are contextual. Identity must support multiple local roles and pseudonymous personas.

Projection over apps

The browser/PWA renders semantic objects. It does not own authority, persistence, or governance.

Explicit governance over pure emergence

Peer networks propagate well, but trust, abuse control, and conflict resolution need defined semantics.

3. Recursive Kernel Definition

Every feature must reduce into the same six primitives.

IrfTek = Object + Authority + Propagation + Projection + Anchor + Expiry
irftek_kernel:
  object:
    required: [id, type, state, authority, visibility]
    purpose: semantic thing

  authority:
    purpose: truth ownership
    rule: closest_valid_authority_wins_locally

  propagation:
    static: ipfs
    live: libp2p_webrtc_gossip
    local: nats_mqtt

  projection:
    human: html_pwa
    ai: semantic_schema

  anchor:
    target: registry_or_ledger
    only_when: public_permanent_or_governance_relevant

  expiry:
    discard: [noise, duplicates, expired_proximity, irrelevant_telemetry]
    compact: [raw_to_event, event_to_state, state_to_summary, summary_to_memory]
Admission test: What object is this? Who owns truth? How does it spread? How is it rendered? What does AI read? What expires? What becomes permanent?

9. AI-Readable Runtime Design

AI reads structured runtime artifacts, not opaque apps.

  • semantic objects
  • authority decisions
  • event streams
  • state tables
  • policy files
  • memory summaries

AI does not scrape
